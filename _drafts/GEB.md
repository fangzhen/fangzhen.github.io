第七章 命题演算
  对话 半音阶幻想曲，及互格 - 对话和标题看不出来有什么关系 - 什么是矛盾？乌龟壳是绿的和乌龟壳不是绿的为什么矛盾？

  构造一个形式系统，包含与、或、非、蕴涵四个词，只依赖正确使用这四个词的那种推理，叫做命题推理。该形式系统的规则：
  > JOINING RULE: If x and y are theorems, then < x∧y> is a theorem.
  > SEPARATION RULE: If < x∧y> is a theorem, then both x and y are theorems.
  > DOUBLE-TILDE RULE: The string '~~' can be deleted from any theorem can also be
  > inserted into any theorem, provided that the result string is itself well-formed.
  > FANTASY RULE: If y can be derived when x is assumed to be a theorem then < x⊃y> is a theorem.
  > CARRY-OVER RULE: Inside a fantasy, any theorem from the "reality" c level higher can be brought in and used.
  > RULE OF DETACHMENT: If x and < x⊃y> are both theorems, then y is a theorem.
  > CONTRAPOSITIVE RULE: <x⊃y> and <~y⊃~x> are interchangeable
  > DE MORGAN'S RULE: <~x∧~y> and ~< x∨y> are interchangeable.
  > SWITCHEROO RULE: <x∨y> and <~x⊃y> are interchangeable.
  其中x 和y必须是良构的串。
  > 问题良构串和形式系统什么关系？

  幻想规则一般称为演绎定理。
  这个系统没有公理，但是幻想规则可以制造定理。例如<P -> ~~P>是定理，P是否是定理不重要，只需要是一个良构串即可。
  岩头之斧：命题演算的一个示例。

  半解释：解释除了原子的其他东西。例如 <P∨~P>半解释为P或者非P。这正是命题演算的关键思想，它产生一些定理，当它被半解释时，被看成是‘普遍为真的半句子’。

  该形式系统下可以用真值表来判定是否是定理。-> 该形式系统是递归的。
  > 但是为什么真值表可以用来判定？从直观上，如果我们接受与或非蕴涵的解释的话，说得通。
  我们知道系统是一致的吗？似乎跟上面的是类似的答案。我们相信符号的意义，就有理由相信是一致的；但更严谨的做法是证明每条定理。
  但是无法证明你在当前系统中的证明是正确的，因为你最终要一直递归地走到上一层的系统。 > 卡罗尔：你永远维护你的推理模式。只能靠信仰了。
  对于导出规则，有类似的问题。导出规则的成立，似乎只能去系统之外去证明。比如已经知道P∨~P是定理，那么我们有理由相信，类似形式的Q∨~Q也是定理，因为我们可以用类似的推导来证明。但是，这个思考过程是形式系统之外的，使用自然语言来证明的。

  推导与证明：这里证明是指人的语言写的，依赖人的思维的证明。 > 可以理解为 证明是以命题演算的形式系统为基础的，但不限于命题演算？
  P∧~P->Q：从一个矛盾出发，结果可以是任何东西。所以， **在命题演算为基础的系统里，是不能包含矛盾的。** < 因此是(内部)一致的？似乎是，因为我们认为结果是任何东西不可接受的。< 第十四章有解释。
  对付矛盾：矛盾的发现促使数学家致力于解决矛盾，而不是认为可以相信任何事情。命题演算系统嵌入到一个更大的系统里去，倘若发现矛盾，意味着这个较大的系统的问题，而不是命题演算的问题，因为命题演算能提供有效的命题推理。> 为什么命题演算可以提供有效的命题推理，以至于问题一定出现在大的系统，而不是命题演算的形式系统中？

第十四章 论TNT及有关系统中形式上不可判定的命题
  对话 蒯恩构造 自指
  哥德尔1931年论文题目 论<数学原理>及有关系统中形式上不可判定的命题。本章题目化用。

  证明对：(m,n)是证明对，当且仅当m是TNT中某个推导的哥德尔数，而该推导的最末一行是哥德尔数为n的符号串。
  对任何形式系统，有两个基本事实：
  基本事实1：“是证明对”这一性质是原始递归的数论性质，因而可以用一个BlooP程序测试。
  基本事实2：“构成证明对”这一性质可以用BlooP测试，因而被一个有两个自由变元的公式在TNT中体现。

  哥德尔第一定理证明：
  TNT定理，比如0=0，可以形式化表示为：∃a: TNT-PROOF-PAIR{a, SSS..SS/a'} 其中S的个数是定理对应的哥德尔数。
  代入关系：三个参数分别是 原公式的哥德尔数 替代自由变元的数 结果的哥德尔数，这三个数是否构成代入关系。它是原始递归的，因此可以在TNT中体现，简写为SUB{a, a', a''}。
  算术蒯恩： ARITHMOQUINE{a'', a'} = SUB{a'', a'', a'}，即把SUB第一个公式的哥德尔数代入自身。ARITHMOQUINE{a'', a'}的解释是：a'是对哥德尔数为a''的公式做算术蒯恩所得公式的哥德尔数，简称为a'是a''的算术蒯恩化。
  构造一个符号串U，把它的哥德尔数记为u。
  ~∃a: ∃a': <TNT-PROOF-PAIR{a, a'} ∧ ARITHMOQUINE{a'', a'}>
  把u代入U的自由变元a''，就得到了哥德尔符号串G：  ~∃a: ∃a': <TNT-PROOF-PAIR{a, a'} ∧ ARITHMOQUINE{SSS..SSS0/a'', a'}>
  1. G的哥德尔数是什么？u的算术蒯恩化
  2. G的解释是什么？不存在数a和a'，使得1)它们形成证明对 2)a'是u的算术蒯恩化。 -> 但是我们知道G的哥德尔数就是u的算术蒯恩化，因而问题出在另一个数a身上，因此G可以解释为：没有一个数能与u的算术蒯恩化形成证明对。 -> 以u的算术蒯恩化为哥德尔数的那个公式不是TNT定理 -> **G不是TNT的定理**
  不完全性论证(第九章有类似论证)：
  G是TNT定理 -> G的解释是真理 -> G不是TNT的定理 -> 矛盾
  G不是TNT的定理 -> G的解释是真理 -> 不矛盾，但可得，至少存在一个不是TNT定理的真理。-> TNT是不完全的。

  哥德尔第二定理
  因为G的解释是真理，~G的解释就为假理。而TNT中推不出假的陈述(因为一致性)，因此~G不是TNT定理。所以G和~G都不是TNT定理。这样，我们在系统中找到一个不可判定的命题G。这可以推出不少结果，如虽然G和~G都不是定理，但公式<G∨~G>是定理。

  这是系统内部的断言与关于系统的断言似乎不太一样的一个简单例子。(个人理解，这是指系统内部我们不知道G是定理，但系统外部，我们知道G的解释是真理)。 - 这使我们怀疑系统是否真的精确地反射了自己。存在于TNT系统内的自我反射的元数学与我们使用的元数学(应该是指非形式系统的元数学)能否很好地相符？哥德尔对在这种自我反射的元数学内证明TNT的一致性很感兴趣，得到了
  哥德尔第二定理：只有在TNT不一致的条件下才能证明TNT的一致性。
  怎么表示TNT的一致性：说不一致是说有两个公式x和~x都是定理，于是所有良构的公式都是定理。因此要证明TNT的一致性，只需要提出一个TNT句子，说明这个句子可以证明为不是TNT定理就可以了。所以表示TNT一致性的一个办法就是说“公式~0=0不是TNT的定理”，其TNT串为‘~∃a: TNT-PROOF-PAIR(a, SSS..SSS0/a')’。其中SSS..SSS0是公式~0=0对应的TNT串。
  可以证明，只要TNT一致，这个句子就不是定理。(本书没有具体证明)
  可以说，TNT表示一些事物时，它的反省能力(introspection)很强，但证明它们时，就很弱了。

  TNT是omega不完全的。对于TNT符号串  ∀a: ~∃a': <TNT-PROOF-PAIR{a, a'} ∧ ARITHMOQUINE{SSS..SSS0/a'', a'}>。它可以直接推出G。如果它是定理，那G也是。但G不是定理，所以它也不是。
  但是与之对应的金字塔公式都是定理：
  ~∃a': <TNT-PROOF-PAIR{0, a'} ∧ ARITHMOQUINE{SSS..SSS0/a'', a'}>
  ~∃a': <TNT-PROOF-PAIR{S0, a'} ∧ ARITHMOQUINE{SSS..SSS0/a'', a'}>
  ~∃a': <TNT-PROOF-PAIR{SS0, a'} ∧ ARITHMOQUINE{SSS..SSS0/a'', a'}>
  ...
  这些定理的解释是0, 1, 2,...与u的蒯恩化(a')不能构成证明对。由于G不是定理，所以没有自然数能与G的哥德尔数构成证明对。所以这些解释每句话都是真的。而由于‘是证明对’这一性质是原始递归的，所以上面每句话都能翻译成TNT定理。也就是说金字塔每一行都是定理，但概述串不是定理。

  在一个给定的系统中，是否可以用哥德尔的自指方法构造一个不可判定的符号串，依赖三个基本条件：
    1. 该系统要足够丰富，以便全部所需要的关于数的陈述，无论真假，都能在其中表示；
    2. 所有的一般递归关系都能用该系统的公式体现；
    3. 公理及根据该系统的规则所确定的印符模式，都能通过某个有限的过程来辨认。做不到这一点，意味着无法区分有效推导和无效推导。

  卢卡斯的论证：无论多么复杂的机器，都对应一个形式系统。接着就能找到一个该系统内不可证的哥德尔公式，而人类知道那是真的。幸亏哥德尔定理，永远是心智说最后一句话。
  从根本上讲，其思想就在于我们总是处于系统之外，从外面我们总能实施哥德尔化运算，以得到程序无法从内部看到的某种真的东西。
  为什么不把‘哥德尔算子’编到程序中去呢？：用以构造哥德尔公式的那个过程是个标准过程，因而确保对每个形式系统能构造出一个哥德尔公式。同样因为是标准过程，那也可以编出程序来执行该过程。而这对应着存在一个具有一条附加推理规则的系统，依次类推。无论多复杂，形式系统都会包含在系统内不可证的公式，而系统之外的人/心智能看出来它是真的。
  但是，抽象地论证能够哥德尔化是一回事，具体搞清在每种情形下如何做是另一回事。

     邱奇-图灵论题不是一个数学定理意义上的一个可证的事实，而是有关人脑中所使用过程的一个假设。
  3. 大众过程形式：假设有一种方法，一个有感知能力的生物可以根据这个方法逐步把数分为两类。又假定这种方法总能在有限时间内得到答案，而且对于给定的数，这种方法总是给出相同的答案。同样假定这种方法可以通过语言由一个有感知能力的生物不走样地传递给另一个有感知能力的生物。那么存在一个有终止的FlooP程序(即一般递归函数)，它给出的答案恰好与这个有感知能力的生物给出的一样。
     这是一种更弱形式的邱奇-图灵论题，只有大众接受的方法(有意识的方法)受FlooP的管辖，对于个人特有的方法并没有给出结论。
     拉马努金(本书翻译拉玛奴衍)是一个证据来反驳标准形式的邱奇论题。
  4. 哈代形式：从本质上讲，所有数学家都同构。
  5. 同构形式：假设有一种方法，一个有感知能力的生物可以根据这个方法逐步把数分为两类。又假定这种方法总能在有限时间内得到答案，而且对于给定的数，这种方法总是给出相同的答案。那么存在一个有终止的FlooP程序(即一般递归函数)，它给出的答案恰好与这个有感知能力的生物给出的一样。而且这个心智过程与这个FlooP程序在下述意义下同构：在某个层次上，大脑与计算机各自执行的那些步骤之间存在一个同构。

  机器的创造性来自于许多层次上硬件与软件的组织与结构。本质上来说，跟人没有区别。人看起来可以改变思想规则，元规则...但说到底，我们改变不了神经元的激发模式。本章的主题就是设想意识如何从错综复杂的神经元中产生。

  任意系统中总存在某个“受保护”的层次，它不会被其他层上的规则所攻击，尽管这些规则之间可能是复杂地缠结在一起的。例如书的作者与书中的角色们；埃舍尔的画手；棋手与可改变规则的棋赛。
  大脑和心智的关系就是神经元缠结支撑着上层的符号缠结。符号缠结使我们产生了意识，造成了一种假象，即“不存在不受干扰的层次”。

  哥德尔定理说明，对具有自我印象的一致的形式系统来说，存在着某些根本性的限制。但这一定理是否具有更普遍的意义？把它当作一个比喻用于心理学或其他学科会具有启发性，但不能直接翻译成其他学科的判断。
  > 个人想法：人脑中的高层次的系统不一定是形式系统，或者即使等价于某个形式系统，也不意味着人脑有限的算力和时间下会受到一致性或完全性的困扰。毕竟大脑的目标不是研究它自己，而是认识这个世界。
  跟哥德尔定理在某种意义上有相似性的其他学科的相关问题：判断自己是否精神正常；我们能否理解自己的心智；个人是否存在；量子力学的测不准原理。
  科学上所有那些本质上依赖与主体与客体聚变的结果都是限制性结果。(> 依赖主体与客体聚变可以理解成自指。)
  在音乐和绘画中，对记号和他们代表的东西之间的区分和联系也有详尽的讨论。偶然音乐；语义错觉画；

  理解心智：怎么定义理解，如果是类似理解汽车工作原理一样理解心智的一般工作原理，似乎哥德尔定理不会对此产生限制。
  人的智能可能达不到理解大脑复杂度的程度，正如螃蟹的智能不足以理解它的大脑一样，但这跟哥德尔定理无关。

  哥德尔定理对于理解心智的启发：一个系统的高层观点可能会包含某种在底层上完全不具有的解释能力。(如果不跳出TNT系统，无法解释哥德尔定理)。
  意识是高层所具有的现象，意识的核心是怪圈，不同层次之间自我强化的共鸣。只是从简化论的观点来看，这过于复杂，但原则上会存在简化论的，及其费解的解释。

  自由意志：代表自我的符号与脑中其他符号相互作用的结果。我们是否认为一个程序是否“在做选择”(有自由意志)的是：我们能否认同于程序运行时所产生的过程的一个高层描述，在一个低层次上，这个程序看起来就像其他程序，但在高层次上，会浮现出意愿、自觉、创造性、意识等词。这与程序是否以确定的方式运行无关。
